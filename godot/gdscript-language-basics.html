GDScript — Language Fundamentals

GDScript is Godot's built-in scripting language: Python-like syntax, indentation-based, dynamically typed by default with optional static typing. If you know Python, you already know ~70% of GDScript — the remaining 30% is engine integration (Nodes, Signals, lifecycle functions). This tool covers the language layer; engine/scene concepts get their own tool.

The Four Foundations
1️⃣
Literal values & variables — how data is stored and named. The building blocks of every program.
2️⃣
Constants — values that never change at runtime; prefer const over var when the value is fixed.
3️⃣
Control flow — if, elif, else, match, for, while. Same logic, different syntax.
4️⃣
Functions & classes — how you organise and reuse code. Master these and switching languages is just memorising syntax.
Quick-Start: Hello World
extends Node

# Called when the node enters the scene tree
func _ready() -> void:
    print("Hello, Godot!")

    # Variables — dynamic (Python-style)
    var score = 0

    # Variables — static (preferred in Godot 4)
    var health: int = 100
    var speed := 5.0   # := infers type

    # Constant
    const MAX_HEALTH = 100

    print("health: %d  speed: %.1f" % [health, speed])
Lifecycle Functions — Execution Order
FunctionWhen it runsCommon use
_init()When the script object is created (before scene)Set default property values; avoid node access here
_enter_tree()Node added to the scene treeRarely needed directly
_ready()Node + all children fully initialised⭐ Your main setup function — safe to access child nodes
_process(delta)Every rendered frameVisual updates, input polling, non-physics movement
_physics_process(delta)Every physics frame (fixed rate, default 60 Hz)⭐ Movement, collision — use this for consistent physics
_input(event)When an input event firesKeyboard / mouse / gamepad events
_exit_tree()Node removed from treeCleanup, disconnect signals
💡 Key Principle from the Series

Master these four foundations — variables, constants, control flow, functions/classes — and switching to any other language (C#, C++, Python, Lua) is just memorising new syntax. The concepts transfer 1:1.

Variable Declaration — Three Styles
# 1. Dynamic (no type) — flexible, less safe
var score = 0
var player_name = "Hala"

# 2. Explicit static type — safest, best autocomplete
var health: int = 100
var speed: float = 5.0
var is_alive: bool = true
var name: String = "Hero"

# 3. Inferred static type  (:=)  — concise + safe
var damage := 25       # inferred as int
var gravity := 9.8    # inferred as float
var tag := "enemy"   # inferred as String

# Constants — SCREAMING_SNAKE by convention
const MAX_HEALTH = 100
const GRAVITY: float = 9.8

# @onready — waits until _ready() before assigning
# (safe way to grab child nodes)
@onready var sprite: Sprite2D = $Sprite2D

Static Typing — Why Bother
⚡
Up to 2× faster — statically typed GDScript compiles to tighter bytecode than dynamic.
🔍
Better autocomplete — the editor knows what methods are available on a typed variable.
🐛
Catch bugs early — type mismatches flagged at edit-time, not at runtime mid-game.
Typing is gradual — mix typed and untyped freely. Start dynamic, add types where it matters.
Export & @onready
# @export — visible in Godot Inspector
@export var speed: float = 200.0
@export var max_health: int = 100
@export var player_name: String = "Hero"

# @onready — assigned after _ready() fires
# $ is shorthand for get_node()
@onready var label := $Label
@onready var timer: Timer = $Timer

# Static variables — belong to class, not instance
static var instance_count: int = 0
String Formatting
var name := "Hala"
var score := 9500

# % format (similar to Python %)
print("Player: %s  Score: %d" % [name, score])

# String concatenation
print("Player: " + name)

# str() conversion
print("Score: " + str(score))

# Multiline strings (triple-quote)
var lore = """
    In the beginning there was Godot,
    and it was free and open source.
"""
⚠️ Gotcha: int ÷ int = int

In GDScript (like Python 2 and C): 5 / 2 == 2, not 2.5. To get float division, make at least one operand a float: 5.0 / 2 or float(5) / 2.

Function Anatomy
# Basic function — no types
func greet(player_name):
    print("Hello, " + player_name)

# Typed function — preferred in Godot 4
# param types after :   return type after ->
func calculate_damage(base: int, multiplier: float) -> int:
    return int(base * multiplier)

# Default parameter values
func heal(amount: int = 10) -> void:
    health += amount

# void = no return value
func reset() -> void:
    health = MAX_HEALTH
    score = 0

# Calling functions
greet("Hala")
var dmg := calculate_damage(50, 1.5)  # returns 75
heal()            # uses default (10)
heal(25)          # overrides default
Lambda Functions
# Lambda — anonymous function stored in a variable
var double = func(x: int) -> int: return x * 2

# Call with .call()
print(double.call(7))   # 14

# Common use: passing to array methods
var nums = [1, 2, 3, 4, 5]
var evens = nums.filter(func(n): return n % 2 == 0)
# evens = [2, 4]

# Lambdas capture surrounding scope
var multiplier = 3
var times_three = func(x): return x * multiplier
Coroutines — await
# await pauses execution until a signal fires
# or a coroutine completes
func _ready() -> void:
    print("waiting 2 seconds...")
    await get_tree().create_timer(2.0).timeout
    print("done!")

    # await a signal from another node
    await $AnimationPlayer.animation_finished

    # await another coroutine
    await some_async_function()
⚠️ A function using await automatically becomes a coroutine — its caller gets a signal back.
Function Best Practices
✅
Use typed signatures — func shoot(target: Node2D, damage: int) -> void catches misuse at edit time and documents intent.
✅
Guard clauses at the top — return early if preconditions fail rather than nesting everything in an if block.
✅
Single return at the end (GDQuest style) — avoids hard-to-track return paths in the middle of functions.
❌
Don't override native methods — Godot 4 warns NATIVE_METHOD_OVERRIDE and treats it as an error. Only override virtual methods (prefixed _).
⚠️
pass keyword — placeholder for empty function bodies, just like Python.
if / elif / else
var health := 45

if health <= 0:
    print("dead")
elif health < 25:
    print("critical")
elif health < 50:
    print("low")          # ← this runs
else:
    print("ok")

# Ternary (inline if)
var state = "alive" if health > 0 else "dead"

# Boolean operators — spelled out like Python
if is_alive and health > 10:
    shoot()
if not is_alive or health == 0:
    die()
match — GDScript's switch
var state := "idle"

# match is more powerful than switch
match state:
    "idle":
        play_idle_anim()
    "run":
        play_run_anim()
    "jump", "fall":   # multiple values
        play_air_anim()
    _:                  # default (wildcard)
        print("unknown state")

# match with type check
match typeof(value):
    TYPE_INT:    print("integer")
    TYPE_STRING: print("string")
    _:           print("other")
for loops
# range(n) — 0 to n-1
for i in 5:            # shorthand
    print(i)            # 0 1 2 3 4

for i in range(5):     # same as above
    print(i)

# range(start, stop, step)
for i in range(2, 10, 2):   # 2 4 6 8
    print(i)

# count down
for i in range(10, 0, -1):
    print(i)            # 10 9 8 ... 1

# iterate array
var enemies = ["goblin", "troll", "boss"]
for enemy in enemies:
    print(enemy)

# break and continue work exactly like Python
for i in 10:
    if i == 3: continue   # skip 3
    if i == 7: break      # stop at 7
    print(i)
while loop
var countdown := 10

while countdown > 0:
    print(countdown)
    countdown -= 1

print("Liftoff!")

# Simulate combat round-by-round
var player_hp := 80
var enemy_hp  := 45

while player_hp > 0 and enemy_hp > 0:
    enemy_hp  -= 12   # player attacks
    player_hp -= 8    # enemy attacks

if player_hp > 0:
    print("Player wins!")
else:
    print("Player lost.")
💡 match vs if-elif chains

Prefer match when checking a single variable against multiple literal values — it's cleaner and more readable than long if/elif chains. if/elif is better when conditions involve comparisons (<, >) or complex boolean logic.

Arrays
# Untyped array — mixed types OK
var bag = ["sword", 42, true]

# Typed array — all elements must be int
var scores: Array[int] = [100, 200, 350]

# Common operations
scores.append(450)          # add to end
scores.push_front(50)       # add to front
scores.pop_back()            # remove + return last
scores.pop_front()           # remove + return first
scores.insert(1, 150)       # insert at index 1
scores.erase(200)            # remove first match by value
scores.remove_at(0)         # remove by index

print(scores.size())         # length
print(100 in scores)         # membership test
scores.sort()                 # in-place sort
scores.shuffle()              # in-place shuffle
var copy = scores.duplicate() # shallow copy

# Functional methods (return new arrays)
var big = scores.filter(func(s): return s > 200)
var doubled = scores.map(func(s): return s * 2)
var total = scores.reduce(func(acc, s): return acc + s, 0)
Dictionaries
# Create — keys can be any type
var player = {
    "name": "Hala",
    "hp": 100,
    "level": 5,
    "skills": ["fireball", "shield"]
}

# Access — bracket or dot notation
print(player["name"])       # "Hala"
print(player.hp)             # 100  (dot notation)

# Safe access (returns null if key missing)
var mp = player.get("mp", 0)  # default 0

# Modify
player["hp"] = 80
player["stamina"] = 50      # add new key
player.erase("stamina")      # remove key

# Check key existence
if player.has("level"):
    print("level: %d" % player.level)

# Iterate — gives keys (NOT key-value pairs!)
for key in player:
    print("%s: %s" % [key, str(player[key])])

# Iterate values only
for val in player.values():
    print(val)

# Typed dict (Godot 4.4+)
var costs: Dictionary[String, int] = {"sword": 50, "bow": 30}
⚠️ Collection Gotchas (especially for Python devs)
⚠️
for dict: gives keys only — unlike Python's for k, v in dict.items(), GDScript gives you the key. Access value via dict[key] inside the loop.
⚠️
Dict == compares by reference — {0:10} == {0:10} is false. Two separate dict literals are different objects even with identical contents. Use .hash() to compare by content.
⚠️
Assignment copies the reference — var b = a does NOT copy an array/dict; both point to the same data. Use a.duplicate() for a shallow copy, a.duplicate(true) for deep.
⚠️
Typed array assign quirk — methods like .filter() and .map() return untyped arrays. Use .assign() to put the result into a typed array variable.
⚠️
No dict.map_in_place() — it doesn't exist. Use a for key in dict: dict[key] = ... loop instead.
⚠️
Don't mutate a collection while iterating it — collect keys/indices to remove first, then erase after the loop.
Class Anatomy
# File: player.gd
class_name Player           # optional: register as global type
extends CharacterBody2D    # inheritance

# --- Signals ---
signal died()
signal health_changed(new_health: int)

# --- Constants ---
const MAX_HEALTH: int = 100

# --- Exported properties (visible in Inspector) ---
@export var speed: float = 200.0

# --- Member variables ---
var health: int = MAX_HEALTH

# --- Onready (assigned after _ready) ---
@onready var anim: AnimationPlayer = $AnimationPlayer

# --- Constructor (rarely needed in Godot) ---
func _init() -> void:
    health = MAX_HEALTH

# --- Lifecycle ---
func _ready() -> void:
    died.connect(_on_died)   # connect own signal

func _physics_process(delta: float) -> void:
    move_and_slide()

# --- Public methods ---
func take_damage(amount: int) -> void:
    health -= amount
    health_changed.emit(health)    # emit signal
    if health <= 0:
        died.emit()

# --- Private / signal callbacks (underscore prefix) ---
func _on_died() -> void:
    queue_free()             # remove node from scene
Signals — The Godot Way
# Define a signal (in the emitter script)
signal enemy_died(enemy_name: String)

# Emit a signal
enemy_died.emit("goblin")

# Connect in code (receiver connects to emitter)
emitter.enemy_died.connect(_on_enemy_died)

# The receiving function
func _on_enemy_died(name: String) -> void:
    print(name + " defeated!")
    score += 100

# Disconnect when done
emitter.enemy_died.disconnect(_on_enemy_died)

# One-shot connection (auto-disconnects)
$Timer.timeout.connect(
    _on_timeout, CONNECT_ONE_SHOT
)
Enums
# Simple enum (auto-increments from 0)
enum State { IDLE, RUN, JUMP, FALL, DEAD }

# Explicit values
enum Direction {
    NORTH = 0,
    EAST  = 90,
    SOUTH = 180,
    WEST  = 270
}

# Usage
var current_state: State = State.IDLE

match current_state:
    State.IDLE: play_idle()
    State.RUN:  play_run()
    State.JUMP: play_jump()

# Enums work as types in static typing
func set_state(s: State) -> void:
    current_state = s
Inheritance
# Parent class: enemy.gd
class_name Enemy
extends CharacterBody2D

var hp: int = 50

func attack() -> void:
    print("Enemy attacks!")

# Child class: goblin.gd
class_name Goblin
extends Enemy              # inherit from Enemy

func _ready() -> void:
    hp = 25               # override property value

func attack() -> void:
    super.attack()         # call parent method
    print("...with a dagger!")

# is keyword — type check
if entity is Goblin:
    entity.attack()

# as keyword — safe cast (returns null on fail)
var goblin := entity as Goblin
if goblin:
    goblin.attack()

Python → GDScript Translation Guide

For Python devs, GDScript is mostly familiar — but there are traps. This tab is the diff, not a full tutorial.

Side-by-Side Syntax Comparison
🐍 Python → GDScript differences
Python
# Booleans
x = True
y = None
not x
x and y

# Print
print("hi", x, sep=",")

# Class init
def __init__(self):
    self.hp = 100

# f-strings
print(f"HP: {hp}")

# Dict iteration
for k, v in d.items():
    print(k, v)

# Comprehensions
evens = [x for x in nums
         if x % 2 == 0]

# Inheritance super()
super().__init__()
GDScript 4
# Booleans — lowercase!
x = true
y = null
not x         # same
x and y       # same

# Print — no sep/end kwargs
print("hi", x)  # space-separated

# Class init
func _init() -> void:
    hp = 100   # no self.

# Format strings — % syntax
print("HP: %d" % hp)

# Dict iteration — keys ONLY!
for k in d:
    print(k, d[k])

# No comprehensions — use .filter()/.map()
var evens = nums.filter(
    func(x): return x % 2 == 0
)
# Inheritance super
super._init()   # or just super()
Python → GDScript Traps
⚠️
true / false / null — lowercase — Python uses True, False, None. GDScript is all lowercase. This will silently fail or error.
⚠️
No self. prefix — Python requires self.hp; GDScript accesses member variables directly as hp. Use self only for disambiguation.
⚠️
No list/dict comprehensions — use .filter(), .map(), .reduce() with lambdas, or a regular for loop.
⚠️
Integer division — 5 / 2 == 2, not 2.5. Write 5.0 / 2 or use float(5) / 2 for float division.
⚠️
print() has no sep/end kwargs — use prints() for space-separated, printt() for tab-separated, printraw() for no newline.
⚠️
Tabs vs spaces — GDScript is strict about indentation consistency. Mix tabs and spaces and you get parse errors.
⚠️
No f-strings — use "value: %d" % x or "value: " + str(x). F-string syntax will throw an error.
⚠️
No tuple unpacking in for loops — for k, v in dict.items() doesn't exist. Iterate keys and access values manually.
Things GDScript Does Better for Games
🎮
Vector math built-in — Vector2, Vector3, Transform2D etc. are first-class types with operators, dot product, cross product, lerp already defined.
🎮
Signals replace callbacks — no need for event systems or observer pattern boilerplate. Signals are a language feature.
🎮
@export puts variables in the Inspector — tweak values without changing code. Faster iteration than Python's argparse or config files.
🎮
await is ergonomic — wait for a timer, animation, or signal in one line, no callback nesting needed.
🎮
match is more powerful than switch — supports multiple values per arm, type checks, and wildcard patterns in one clean construct.
