Step-by-Step Walkthrough
1
Find Inspiration
Go to awwwards.com (Awards — three W's) or similar frontend showcases. Find a site you can imagine templating and making your own. Look for design techniques you've never seen before — parallax, scroll animations, shadowing effects, etc. 💡 Awards shows you how pros actually build frontends — not the 10 millionth version of a boring SaaS site.
2
Site Teardown — Extract the Source
Give Claude Code three things: the HTML (press Ctrl+U on the page, select all, copy, paste), the page URL, and screenshots of key effects. Then use the site teardown skill so Claude Code can fetch and fully analyze the CSS and JavaScript files found in the HTML. ⚠ Don't rely on screenshots alone — without the JS and CSS, Claude Code won't understand how effects actually work.
3
Asset Creation — Build Your Visual Raw Material
Find or generate any bespoke images/assets. For animated assets (like a shading moon), generate a start-state and end-state image, create a short transition video using an AI video tool, then have Claude Code extract all frames from the video. Each frame corresponds to a state in the animation. 💡 Always generate on a green screen so AI can cleanly remove the background across all extracted frames.
4
The Build — Hand It All to Claude Code
By now Claude Code has done its teardown analysis. Hand it your MP4 (or extracted frames), your other assets, and say: get to work. It will implement the effect stack it identified — GSAP, ScrollTrigger, Lenis, or whatever the original used. Review its breakdown; that's where you learn. ✓ Claude Code will name the techniques it sees. Look them up. This is front-end education happening in real time.
5
Iteration — Make It Yours
Compare screenshots of your version vs. the original. Feed the diff back into Claude Code ("here's mine, here's the original, what's different?"). Once you're happy with fidelity, swap in your own assets, headlines, colors, and copy. You're no longer cloning — you're building. 💡 Swap the scroll animation subject for something relevant to your project. The technique transfers; the asset is yours to replace.

⚠ Web Fetch Limitation

Web fetch delegates to a smaller sub-model. Large JS/CSS files will be partially dropped with a plain fetch. The skill's dynamic prompts are specifically built to work around this — they target class names from the HTML so only relevant code is returned.

✓ Teardown Output

The skill saves a structured markdown file to research/YYYY-MM-DD-{slug}-teardown.md — tech stack, effects breakdown, design system, assets list, and section-by-section build plan.

How the Skill Works Internally

The skill runs a 5-step pipeline: get HTML → find JS/CSS paths → fetch JS with dynamic prompt → fetch CSS with dynamic prompt → assemble output document. The dynamic prompting (using class names from the HTML) is what separates it from a plain web fetch.

1
Get the HTML
If you pasted raw HTML — use it directly (more complete than a fetch for large pages). If only a URL was provided, fetch with a prompt asking for full class names, element hierarchy, image/video paths, SVGs, script paths, and inline styles. Don't summarize. 💡 Pasted HTML always wins over fetched HTML. Ctrl+U → Ctrl+A → Ctrl+C on the page.
2
Find JS & CSS File Paths
Scan the HTML for <link rel="stylesheet"> and <script src> tags. Construct full URLs from domain + path. Skip all third-party scripts (GTM, analytics, consent tools, CDN-hosted libraries — note what they are, don't fetch them). 💡 Target files in /dist/, /build/, /assets/, /static/ with names like main.js, app.css. Also check <link rel="modulepreload">.
3
Fetch the JavaScript — Dynamic Prompt
Build the fetch prompt by taking the base JS extraction prompt and appending the actual class names found in the HTML. The prompt asks for: all animation code, ScrollTrigger configs, mouse/scroll handlers, GSAP timelines, IntersectionObserver, CSS custom property manipulation, cursor code, preloaders, image sequences — with actual code snippets, not summaries. ⚠ Focus on classes with words like: hero, animate, scroll, slide, reveal, split, parallax, hover, frame, cursor, loader, marquee, zoom.
4
Fetch the CSS — Dynamic Prompt
Same dynamic approach: take the base CSS extraction prompt and append class name stems from the HTML. Asks for: every color value, @font-face declarations, @keyframes, CSS custom properties, media query breakpoints, transforms, z-index stacking, spacing system, blend modes, noise/grain overlays, mask-image, clip-path — actual CSS rules, not summaries. 💡 Use stems not full names: if the HTML has .home-hero__moon-inner, pass hero, moon to the prompt.
5
Assemble the Teardown Document
Combines all three sources into a structured markdown file saved to research/. The document is thorough enough that someone could build a clone in a fresh Claude Code session without ever visiting the original site.

Quality Signal

A good teardown is specific, not vague. "GSAP ScrollTrigger with scrub:true pinned to .hero, scaling from 1 to 0.3" is useful. "Uses scroll animations" is not. The document should also distinguish confirmed (found in source) vs inferred (guessed from class names).

The Core Insight

Award-winning visual effects almost always have simple underlying implementations. The teardown reveals this. Here are the patterns you'll encounter most often.

Effect (what it looks like)Implementation (what it actually is)
Video / 3D object reacting to scroll or mouse Preloaded image frames — swap visibility based on input position. Just images.
Characters flying in one by one GSAP SplitText wraps each character in a span, staggers opacity/transform.
Elements moving at different speeds on scroll Multiple elements with different multipliers on transform — parallax layers.
Animations tied precisely to scroll position GSAP ScrollTrigger with scrub: true. Binds any animation to scroll progress.
Complex CSS-driven effects that JS controls JS updates a --progress CSS variable; CSS uses calc() with it.
Buttery smooth scrolling Lenis or Locomotive Scroll library. One init call.
Page-to-page transitions Barba.js or Swup — overlay fades in, content swaps, overlay fades out.
Film grain / noise texture overlay Fixed div with noise texture image + mix-blend-mode: overlay.
Custom cursor that tracks smoothly GSAP quickSetter tracking mouse position on a fixed element.
Infinite scrolling text marquee CSS or GSAP infinite horizontal translation loop.
Button that subtly follows your cursor Mousemove + transform on the button — magnetic button effect.
Elements that fade/slide in when scrolled to IntersectionObserver or ScrollTrigger adds a class; CSS handles the transition.
✓ Use This List

When you see an effect on an award site and can't figure out how it works, check this list first. The teardown will confirm it — but chances are it's one of these twelve patterns dressed up with good design.

Step 3 — Asset Creation

Bespoke sites use bespoke assets. Claude Code can't conjure a custom moon or cocktail glass — you need to create or source those visuals. The technique below handles animated assets (the scroll-animation subject) using a video-to-frames workflow.

The Video-to-Frames Technique

The original Moon site loaded 54 individual frame images for its shading animation. Instead of generating 54 images, generate two + a video transition, then extract the frames.

🌑
Start Image
Asset fully illuminated — on green screen
🌕
End Image
Asset fully shadowed — on green screen
🎬
Video Transition
AI video tool (e.g. Veo 3.1) animates between states
🖼️
Frame Extraction
Claude Code strips every frame from the MP4
🌐
Animation
Mouse position / scroll maps to a frame index
Image Generation Tips
💚
Always use green screen. The asset needs to be isolated from its background across every extracted frame — green screen makes AI removal easy.
📸
Take screenshots from the original as reference. Screenshot the fully lit state and the fully shadowed state, then pass them to your image generator with: "remove all text and make the background a green screen."
🔧
Tool suggestion: NanoBanana Pro for image generation. Google Veo 3.1 (or similar) for the video transition.
Video Generation Warnings
🚫
No camera movement. Explicitly prompt: no zoom, no drift, no rotation. If the asset changes size across frames, it will look broken when mapped to mouse position.
⏱️
Keep it short — 4 seconds max. 8 seconds = hundreds of frames = too much to load and too slow. 4 seconds gives you ~50–60 usable frames.
🔁
Expect to run it several times. Video generation is more inconsistent than image generation. Find one you like, then download it.
Exact Video Prompt Structure
# Key instructions to include in your video generation prompt:

Static shot — camera doesn't move
Shadow sweeps across the surface following the 3D contours
No camera movement
No zoom
No drift
No rotation

# Repeat the no-movement constraint multiple times in your prompt.
# AI video models will drift or zoom unless you're explicit.

ℹ How the Animation Works

Each mouse position (or scroll position) maps to a specific frame number. The original Moon site used 54 frames. Your extracted video frames serve the same role — load them all, then show the frame that corresponds to the current input position.

Gotchas
⚠️
Screenshots alone ≠ a good build. Claude Code needs the actual JS and CSS — not just how things look, but how they're built. Always run the teardown skill.
⚠️
Plain web fetch is incomplete. The sub-model Claude Code uses for fetching won't return large JS/CSS files in full. The teardown skill's enhanced prompt is specifically designed to work around this.
⚠️
Camera movement in video kills the effect. Any zoom, drift, or rotation means the asset changes size/position across frames — it will look broken when mouse-mapped. No exceptions.
⚠️
Too many frames = slow load. Keep video to 4 seconds. 8 seconds generates hundreds of frames that will kill page load performance.
⚠️
No green screen = messy frame extraction. AI background removal across dozens of frames only works cleanly when the source background is pure green. A complex background will produce inconsistent cutouts.
⚠️
Don't stop at the clone. If you just copy the site and never swap the content, you missed the whole point. The output should eventually look nothing like the original.

1
Side-by-side screenshot compare
Take a screenshot of your build. Take a screenshot of the original. Notice the differences: text sizing, spacing, proportions, animation timing.
2
Feed the diff back to Claude Code
Paste both screenshots into Claude Code and say: "Here's my version. Here's the original. What's different in the code? Let's line those up."
3
When it's close enough — make it yours
Swap out the scroll-animation asset for something relevant to your project. Change the headline. Update the color palette. The scroll-animation technique stays; everything else becomes original. ✓ After this step, you own a unique site that no AI template could have produced.

✅Keep video transitions to 4 seconds or less

✗ Avoid These

❌Generating video longer than ~4 seconds for frame assets
❌Any camera movement in the video generation prompt

❌Using complex backgrounds when green screen is possible
