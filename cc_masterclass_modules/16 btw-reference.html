/btw — Side Questions Without Breaking Focus
REFERENCE
DARK

/btw — by the way

Ask Claude a quick question while it's working — without stopping the task, burning context, or opening a second terminal. The answer appears in a dismissible overlay and leaves zero footprint on your conversation history.

What Makes It Special
🔍
Full context access — sees everything Claude has read, every decision and edit so far in your session.
🚫
No tools, no side effects — can't run commands, read new files, or edit code. Read-only by design.
🧹
Zero history footprint — the question and answer never enter your conversation. Dismiss and it's gone.
⚡
Non-blocking — fire it while Claude is actively mid-task. The main process keeps running.
1️⃣
One question, one answer — no back-and-forth. Use a normal prompt if you need a real conversation.
Cost Benefit
$0
Added to context window cost
↓
Context growth per side question
/btw reuses the parent conversation's prompt cache, so the marginal cost is minimal. Your throwaway question doesn't permanently inflate the context window — and every future turn in that session benefits.
/btw vs Sub-Agent — The Opposite Tradeoff
Capability /btw Sub-Agent
Parent context visible ✓ Full ✗ Empty
Can run tools / commands ✗ None ✓ Full
Answer saved to history ✓ No — ephemeral ✗ Yes
Best for Recalling something Claude already knows Doing something independently
Both tools share the same goal: keeping your main conversation clean. They just approach it from opposite ends.
💡 Mental Model

/btw is the sticky note on your monitor. It knows everything happening on the screen, costs nothing to write, and you throw it away once you've read it.

How to Use /btw — Step by Step
1
Type /btw followed by your question
Works while Claude is mid-task or between prompts. No special setup required — available in Claude Code v2.1.72+. 
/btw what was the name of that database migration file?
/btw which file has the auth middleware?
/btw what port is the dev server running on?
2
Read the answer in the overlay
Claude responds in a dismissible overlay on top of your current session. The main task keeps running in the background. 💡 Copy or note the answer if you need it — once dismissed, it's gone for good.
3
Dismiss with any of these keys
Space Enter Escape
The overlay disappears. Nothing was added to your conversation history. You're right back where you were.

⚠️ Won't Work

Asking about a file Claude hasn't read yet — /btw only sees what's already in the conversation. It can't open new files on your behalf.

✓ Great For
✅Recalling something Claude already read or saw
✅Quick questions during a long-running task (40+ min)
✅Understanding a decision Claude made without interrupting it
✅Progress checks mid-refactor
✅Throwaway questions you don't want in history
✗ Wrong Tool
❌Files Claude hasn't opened yet — it can't look those up
❌Answers that need web search, docs, or code execution
❌Follow-up conversations — one question, one answer only
❌Redirecting Claude's current plan — use a normal prompt
❌Cleaning up already-accumulated context — use /compact or /clear
Gotchas
⚠️
Answer disappears on dismiss: there's no way to retrieve it. Copy or note the answer before pressing Space / Enter / Escape.
⚠️
Context boundary is the conversation, not the filesystem: /btw knows exactly what your main session knows — no more, no less. Unread files are invisible to it.
⚠️
/btw is invisible to the main task: Claude's active work won't be affected by your side question. If you want to change direction, use a normal interrupt prompt.
⚠️
Not a substitute for /compact or /clear: /btw prevents individual questions from entering history, but it won't compress context that's already piled up.
⚠️
Single-response only: no back-and-forth. If you need to dig deeper, wait for the current task to finish or use a normal prompt.
✅ Complements /btw

/btw + sub-agents are a powerful pair. Use sub-agents when you need Claude to independently go DO something (read files, run commands). Use /btw when you need Claude to RECALL something it already knows. Different tools, same goal: keeping your main conversation clean.
