💡 Why These Three

Each repo plugs a different gap: Codex Plugin = second-opinion reviews without leaving Claude Code. NotebookLM PI = unlock research automation the web UI won't let you batch. Obsidian Skills = ground Claude in your local vault with zero hallucination on format.

What It Does
🔍
/codex:review — Runs a standard Codex code review on your current work; same quality as running /review inside Codex directly.
⚔️
/codex:review --adversarial — Skeptical review mode. Point it at code built by Opus and get pushback from an independent model.
🤖
/codex:rescue — Delegate a task to Codex as a subagent. Supports --background, --wait, --resume, --fresh. Returns a session ID you can resume later inside Codex.
📊
/codex:status / /codex:result / /codex:cancel — Manage running or finished background Codex jobs.
🔧
/codex:setup — Checks whether Codex CLI is installed and authenticated. Can offer to install it via npm. Also controls the optional review gate (stop hook that blocks Claude exit until Codex review passes).
Setup Requirements
🔑
Auth: ChatGPT subscription (Free tier works) or OpenAI API key. Run !codex login if not already signed in.
📦
Install the plugin: Claude Code marketplace. Existing Codex users get zero extra config — plugin re-uses your local CLI auth.
⚙️
Custom default model/effort: Add to .codex/config.toml at vault root. E.g. model = "gpt-5.4-mini".
Gotchas
⚠️
Read-only: /codex:rescue does NOT auto-fix code. It investigates and reports — Claude still applies fixes.
⚠️
Review gate loop risk: Enabling the stop hook (--enable-review-gate) can create a long Claude/Codex loop and drain usage limits quickly.
⚠️
Multi-file review is slow: Run multi-file reviews in --background mode. Use --base <ref> for branch-scoped reviews.
Example Commands
# Standard code review
/codex:review

# Adversarial review (skeptical mode)
/codex:review --adversarial

# Delegate investigation to Codex in background
/codex:rescue --background investigate why the tests started failing

# Resume a previous rescue session
/codex:rescue --resume apply the top fix from the last run

# Use a specific model + effort level
/codex:rescue --model gpt-5.4-mini --effort medium investigate flaky test

# Check setup status
/codex:setup
🎯 Best Use Case

After building something significant with Claude Opus, run /codex:review --adversarial to get an independent, skeptical second opinion before merging. It's the closest thing to a peer code review loop in an AI-only workflow.

Gotchas
⚠️
Unofficial API: Uses undocumented Google RPC calls that can break without notice if Google changes internals.
⚠️
Auth sessions expire (~20 min). If commands fail with auth errors, re-run notebooklm login.
⚠️
Use a dedicated Google account for automation — browser humanization is built in but detection risk is real (like web scraping).
⚠️
CSRF tokens expire: Use client.refresh_auth() or re-run login when API calls start failing.

The 5 Skills (kepano/obsidian-skills)
📝
obsidian-markdown — Core skill. Teaches agents Obsidian-flavored Markdown: [[wikilinks]], callouts, frontmatter/YAML, tags, embeds (![[file]]). Without this, Claude breaks wikilink syntax.
🗃️
obsidian-bases — Handles .base files (Obsidian's database layer). Without it, Claude generates invalid JSON for Base files.
🗺️
json-canvas — Handles .canvas files for spatial note maps / whiteboards. Without it, canvas output won't open in Obsidian.
🌐
defuddle — Strips web pages to clean markdown before saving. Removes ads, nav bars, footers before content hits Claude. Cuts token cost significantly on web research sessions.
⌨️
obsidian-cli — Teaches Claude the Obsidian CLI commands. Lets Claude interact with your vault directly from terminal (search, open, create, tag operations).
