9 High-Leverage Claude Code Tips

Dead-simple, high-impact techniques that vault you past most Claude Code users. Every tip here trades minimal setup cost for disproportionate daily gains in speed, token efficiency, and output quality.

TIP 01
CLI over MCP
Swap MCPs for CLI tools when a terminal interface exists. Fewer tokens, less overhead, often more functionality. Pair with skills.
$ claude-tool --flag
TIP 02
/btw — Sidebar Chat
Ask a side question mid-task without touching the context window. Full conversation visibility, ephemeral answer, zero history cost.
/btw What does X return?
TIP 03
/hook — Sound Alert
Play a subtle ding when Claude finishes. Stop babysitting multiple windows. Saves hours per week through better attention management.
/hook play sound on stop
TIP 04
/clear — Early & Often
Reset the context window before it degrades. At 1M tokens, Claude Opus accuracy drops to 65%. Clear at 20–25% for best results.
/clear
TIP 05
/statusline — Stay Oriented
Always-visible status bar showing directory, model, and context %. No more guessing. Set it up once with a single command.
/statusline
TIP 06
Skill Creator
Anthropic's most powerful skill. Creates, modifies, and benchmarks other skills with quantifiable scores. The meta-skill for skill management.
skill-creator
TIP 07
Agent Teams
Experimental: sub-agents that talk to each other, not just to Claude. Parallel, coordinated work. Enable via env var, invoke explicitly.
CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
TIP 08
Open-Ended Plan Mode
Ask "What am I not thinking about?" in plan mode. Forces deeper questions, surfaces unknown unknowns, and educates you along the way.
plan mode → open questions

Claude + Obsidian
Open Claude Code inside an Obsidian vault. Claude writes .md files, Obsidian gives you a visual knowledge graph. Free, symbiotic setup.
cd ~/vault && claude
💡 Core Philosophy

Every tip here shares two traits: they're high leverage and dead simple. You don't need all nine at once — pick whichever addresses your biggest current friction and implement it today.

Context Window Hygiene

The more tokens you accumulate, the worse Claude gets. Understanding context rot and building habits around it is the single biggest performance lever in Claude Code.

Context Rot — The Numbers
91%
Accuracy at 256k tokens
65%
Accuracy at 1M tokens (Opus)
Claude Sonnet drops even further — to 78.3% at 1M. These numbers come from Anthropic's own long-context retrieval benchmarks. The degradation is not a bug; it's physics. Manage around it.
The 20–25% Rule
📊
Clear whenever you hit 20–25% context usage — unless you have a specific reason to preserve history.
🔄
/clear resets the context window entirely. Input history resets too, but the prior session is preserved for /resume.
💊
/compact compresses instead of clearing — useful mid-task when you can't afford a full reset.
👁️
Use /context at any point to see current usage. Or just set up the status line (Tip 05) for always-on visibility.
TIP 05 — /statusline Setup
1
Run the built-in command
Type /statusline in any Claude Code session and describe exactly what you want displayed. 💡 Example prompt: "Show my directory, current model, and context window % at all times."
2
Claude writes the script
It creates ~/.claude/statusline.sh, makes it executable, and wires it into ~/.claude/settings.json automatically. 
# settings.json entry Claude adds:
"statusLine": { "type": "command", "command": "~/.claude/statusline.sh" }
3
The status line refreshes on every message
Available data fields from the JSON stdin: model.display_name, context_window.used_percentage, workspace.current_dir, cost.total_cost_usd, rate_limits.five_hour, worktree.name ⚠ Values may be null before the first API response — add // 0 fallbacks in jq. 
# Minimal bash statusline script (~/.claude/statusline.sh)
input=$(cat)
MODEL=$(echo "$input" | jq -r '.model.display_name')
PCT=$(echo "$input" | jq -r '.context_window.used_percentage // 0' | cut -d. -f1)
DIR=$(echo "$input" | jq -r '.workspace.current_dir')
echo "[$MODEL] 📁 ${DIR##*/} | ${PCT}% ctx"

Slash Commands & CLI-First

Three of the nine tips live entirely in slash commands. The philosophy: anything that can be a CLI should be a CLI — Claude Code is a terminal tool and there's zero overhead for native shell execution.

TIP 01 — CLI over MCP
✓ Prefer CLI
✅Tool has a terminal interface? Use the CLI directly.
✅Always pair a new CLI with a skill that teaches Claude how to use it.
✅Native shell calls = no protocol overhead, lower token costs.
✅CLIs often expose more functionality than their MCP wrapper.
✗ Keep MCP for...
❌Services with no CLI (web APIs, SaaS platforms only).
❌Auth-heavy integrations where MCP handles credentials cleanly.
❌Cases where the MCP skill already works perfectly.
TIP 02 — /btw (By The Way)
💬
What it does: Spawns an ephemeral side query with full session context. The answer appears in a dismissible overlay. Zero history impact.
🔒
Read-only mode: /btw has no tools — it can't run commands or edit files. It's purely a knowledge lookup against your current session.
⚡
Token cost: Minimal — reuses the parent conversation's prompt cache. Side question + answer never enter the main thread's token count.
🔄
Works while Claude is running: You can fire /btw mid-task without interrupting the main agent loop. Available since Claude Code v2.1.72 (March 10, 2026).
🆚
vs Subagent: /btw sees your full conversation but has no tools. Subagents have full tools but start with empty context. Use /btw to ask about what Claude already knows; use a subagent to go find something new.
Dismiss Keys

Press Space, Enter, or Escape to dismiss the /btw overlay and return to your prompt.

TIP 03 — /hook Sound Notifications
1
Quick setup via slash command
Run /hook and say "Play a sound when you complete a task." Claude configures the Stop hook automatically. 💡 You can point it at any audio file, or use system sounds.
2
Manual setup via settings.json
Add to ~/.claude/settings.json: 
"hooks": {
  "Stop": [{ "matcher": "", "hooks": [{ "type": "command",
    // macOS:
    "command": "afplay /System/Library/Sounds/Glass.aiff"
    // Linux: "aplay ~/done.wav"
    // Windows: "powershell ... Media.SoundPlayer ..."
  }]}]
}
3
Hook event types
Stop — main task complete · SubagentStop — background task done · Notification — Claude needs your input / permission prompt ⚠ Stop fires when Claude finishes responding, not on user interrupts. Use & to run in background so sound doesn't block Claude.

Agents & Skills

Two of the most powerful (and underused) features: Skill Creator for meta-skill management, and Agent Teams for true parallel multi-agent coordination.

TIP 06 — Skill Creator (Anthropic's Meta-Skill)
🏗️
Creates skills: Guides you through building a new SKILL.md from scratch — asks for concrete examples, identifies reusable resources, generates the frontmatter.
✏️
Modifies skills: Iterates on existing SKILL.md files based on feedback. Understands what another Claude instance needs (not what you want to say).
📊
Benchmarks skills: Runs parallel test cases against your skill, grades outputs, generates an HTML eval viewer. Gives you actual quantifiable numbers before and after changes.
📦
Packages skills: Validates and zips the skill directory into a distributable file, maintaining the proper structure for sharing.
Skill Directory Structure

Every skill is a folder with at minimum a SKILL.md file containing YAML frontmatter (name, description) and markdown instructions. Optional: scripts/, references/, assets/ subdirectories.

TIP 07 — Agent Teams (Experimental)
Enable
# Option A: Environment variable
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude

# Option B: settings.json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
Invoke
# Must be EXPLICIT — it won't auto-spawn
# Wrong: just ask it to do parallel work
# Right:
"Create an agent team to..."
  one teammate on UX
  one on technical architecture
  one playing devil's advocate
🆚
vs Sub-agents: Normal sub-agents only report back to the main Claude. Agent teammates can message each other directly, share a task list, and coordinate without the lead as intermediary.
🏗️
Architecture: One session = team lead. Teammates each have their own 1M-token context window. Communication via a shared mailbox system at ~/.claude/teams/{team}/inboxes/.
✅
Best for: Research from multiple angles · New modules (each teammate owns a piece) · Debugging competing hypotheses · Cross-layer work (frontend / backend / tests).
❌
Avoid for: Sequential tasks with dependencies · Same-file edits · Simple tasks where overhead isn't worth it. Costs significantly more tokens than a single session.
⚠️
Known limitations: No session resumption for in-process teammates · Task status can lag · Shutdown can be slow · One team per session · No nested teams. Requires v2.1.32+.

Workflow Upgrades

Tips 08 and 09 are about how you think with Claude Code, not just what commands you run. One is a mindset shift, the other is a tool pairing that turns Claude Code into a genuine second brain.

TIP 08 — Open-Ended Questions in Plan Mode
🧠
The core problem: If you're non-technical, you don't know what you don't know. Plan mode's default questions are surface-level. Push deeper.
❓
High-leverage prompts in plan mode:
  • "What am I not thinking about?"
  • "What are the unintended consequences?"
  • "What would an expert in [domain] be thinking about right now?"
  • "What assumptions am I making that might be wrong?"
📈
Why it works: Forces Claude to surface domain-specific risks and edge cases it would otherwise skip. You get better outputs and educate yourself in the process.
TIP 09 — Claude Code + Obsidian
1
Install Obsidian (free)
Download from obsidian.md. Create a vault — just a designated folder anywhere on your computer.
2
Open Claude Code inside the vault
cd ~/vault   # or wherever your vault lives
claude
That's the entire setup.
3
Have Claude write markdown files
Tell Claude Code to save all notes, plans, and documentation as .md files in Obsidian format. Obsidian auto-indexes them. 💡 Obsidian's graph view shows how all your Claude-generated docs connect to each other.
4
The symbiotic loop
Claude builds files in the terminal → Obsidian renders and links them → You navigate visually → You pass context back to Claude. Best for projects with large, growing bodies of text (personal assistant, research, docs) — not traditional code projects.
⚠ Obsidian Best Fit

The Obsidian pairing shines when there's a massive and growing body of markdown files. Think personal assistant workflows, research projects, or documentation-heavy products — not standard coding projects where a file tree works fine.

Gotchas & Common Mistakes
⚠️
Agent Teams won't spawn automatically. You must enable the env var AND explicitly say "create an agent team." Neither step alone is enough.
⚠️
/btw answers disappear when dismissed. They are not saved to history. If you get a useful answer, copy it before pressing Escape.
⚠️
Sound hooks need & to run in background. Without the ampersand, afplay blocks Claude until the sound finishes playing.
⚠️
Statusline values can be null early in a session. Add // 0 fallbacks in all jq expressions. The script fails silently if null isn't handled.
⚠️
CLI tools need a companion skill. The CLI is just an executable. Without a skill that teaches Claude its interface, usage flags, and best patterns, it won't use it well.
⚠️
Agent Teams have session resumption issues. /resume and /rewind do not restore in-process teammates. If a team goes missing after resume, spawn new teammates manually.
