💡 Core Idea

Every software becomes instantly agent-controllable through structured CLI. Agents control any application without APIs, GUIs, rebuilding, or complex wrappers. One command transforms human-designed software into agent-native tools.

The 7-Phase Pipeline

Fully automated — you point it at a repo and walk away. Takes ~20 minutes for a medium-sized app.

1Analyze
2Design
3Implement
4Plan Tests
5Write Tests
6Document
7Publish
Phase Breakdown
1
Analyze — Scans source code, maps GUI actions to underlying APIs and backend calls. ⚠ This is why open source is required — it needs to see the actual source.
2
Design — Architects command groups, state model, and output formats (JSON + human-readable).
3
Implement — Builds a Click CLI with REPL interface (repl_skin.py), --json flag on every command, and undo/redo support.
4
Plan Tests — Creates TEST.md with unit and E2E test plans.
5
Write Tests — Implements the test suite: unit tests with synthetic data + E2E tests against real software backends.
6
Document — Updates TEST.md with results. Auto-generates a SKILL.md inside the package for agent discovery (Phase 6.5). 💡 The REPL banner displays the SKILL.md absolute path at startup — agents auto-discover it.
7
Publish — Creates setup.py, installs CLI to PATH as cli-anything-<software>. 💡 Completion message: "All success criteria are met."
Core Design Principles
🔌
Authentic integration. CLI generates valid project files and calls the real application for rendering. No toy implementations, no Pillow replacements for GIMP.
🔁
Dual interaction modes. Every CLI runs as a stateful REPL (enter bare command) or as a subcommand pipeline for scripting agents.
🤖
Agent-native by default. --json on every command. Agents discover capabilities via --help and which.
🚫
Zero graceful degradation. Tests fail (not skip) when backends are missing — guarantees authentic functionality is present.
🔑 What "Agent Harness" Means

Claude Code calls the generated CLI a "CLI wrapper" or "agent harness." This is just the name for the installable Python package under cli_anything.<software> that wraps the real app's backend with a clean CLI surface.

Prerequisites
🐍
Python 3.10+ required. (Video mentions 3.1 — repo requires ≥ 3.10.)
💻
Target software installed. The real app must be on your machine — CLI Anything calls its actual backend.
🪟
Windows users: Claude Code runs via bash. Install Git for Windows (includes bash + cygpath) or use WSL — otherwise commands fail.

Using the Generated CLI
1
Install to PATH 
cd drawio/agent-harness && pip install -e .
2
Subcommand mode — for scripts and agents 
cli-anything-drawio --help
cli-anything-drawio --json diagram new --name MyArch
3
REPL mode — for interactive sessions 
cli-anything-drawio          # enter bare command → REPL
drawio> diagram new --name MyArch
drawio> exit
4
Refine if coverage is incomplete 
# Broad gap analysis
/cli-anything:refine ./drawio

# Targeted refinement
/cli-anything:refine ./drawio "batch export and SVG embedding"
💡 Refine is incremental and non-destructive — run it multiple times.

Critical Lessons from HARNESS.md
⚠️
Use the real software. The CLI must call the actual application for rendering. No substitute libraries. Generate valid project files → invoke the real backend.
⚠️
The Rendering Gap. GUI apps apply effects at render time. If your CLI manipulates project files but uses a naive export tool, effects are silently dropped. Fix: native renderer → filter translation → render script.
⚠️
Timecode precision. Non-integer frame rates (29.97fps) cause cumulative rounding errors. Use round() not int() and allow ±1 frame tolerance in tests.
⚠️
Output verification. Never trust exit code 0. Verify: magic bytes, ZIP/OOXML structure, pixel analysis, audio RMS levels, and duration checks.
⚠️
Filter translation. When mapping effects between formats (MLT → ffmpeg), watch for duplicate filter merging, interleaved stream ordering, and unmappable effects.
