Remote Control — Claude Code
REFERENCE
DARK

Control Claude Code from anywhere

Remote Control bridges your local Claude Code terminal session with claude.ai/code and the Claude mobile apps. Start a task at your desk, pick it up from your phone — your code never leaves your machine.

What it does
📱
Control your local Claude Code session from your phone, tablet, or any browser.
🔒
Code stays local. Only the conversation syncs — your filesystem, MCP servers, tools, and project config stay on your machine.
🔁
Full two-way sync. Send messages from terminal, web, and phone interchangeably. Everything stays in sync.
🔌
Auto-reconnects. If your laptop sleeps or network drops, the session reconnects automatically.
🛡️
Secure by design. Outbound HTTPS only — no inbound ports opened. Traffic goes through Anthropic API over TLS with short-lived, scoped credentials.
Requirements
Pro / Max / Team / Enterprise v2.1.51+ claude.ai login
💳
API-only keys don't work. You must be on a Pro, Max, Team, or Enterprise plan and authenticated through claude.ai.
📦
Claude Code v2.1.51 or later. Check with claude --version.
🔑
Logged in via /login inside Claude Code (claude.ai auth, not API key).
📂
Run claude in your project directory at least once to accept the workspace trust dialog.
💡 Mental model

Think of it like a shared screen for your Claude session — but instead of streaming pixels, it's streaming the conversation. Your terminal is the actual engine. The phone or browser is just the remote control.

Step-by-Step Setup
1
Check your requirements
Confirm you have a Pro/Max/Team/Enterprise plan (not API-only), Claude Code v2.1.51+, and are logged in via claude.ai auth. 💡 Run claude --version to check version. Run /login inside Claude Code if not yet authenticated.
2
Accept workspace trust (first time)
Navigate to your project directory and run claude at least once. Accept the trust dialog that appears.
3
Start a Remote Control session
From your project directory, run: 
claude remote-control

# Optional: give it a name for easy identification
claude remote-control --name "My Project"
The terminal shows a session URL and a QR code. Press spacebar to toggle the QR code on/off.
4
Connect from another device
Three ways to connect once the session is running: 
Option A: Scan the QR code with your phone camera → opens in Claude app
Option B: Copy the session URL → paste into any browser
Option C: Visit claude.ai/code → find session in list (green dot = online)
5
Enable from inside an existing session
Already in Claude Code and want to add remote access without restarting? 
/remote-control          # enable remote control
/rc                      # shortcut
/remote-control My Project  # with a custom name
💡 This generates the session URL and QR code without interrupting your current work.
6
Enable for all sessions (optional)
Want every Claude Code session to be remotely accessible by default? 
/config   # then set "Enable Remote Control for all sessions" → true
Now every time you run claude, it automatically registers as a remote session.
✅ You're set when...

You can see the session URL in your terminal and a green status dot next to your session name on claude.ai/code. Messages sent from your phone appear in the terminal.

Pattern 1: Desk to Couch
→
The scenario: Working at your desk, need to step away but want to keep an eye on things or send follow-up prompts from your phone.
→
How: 
claude remote-control --name "Feature Build"
Scan the QR code with your phone. Walk away. Send messages, review diffs, and approve actions from the Claude app while Claude keeps working on your machine.
Pattern 2: Long-Running Tasks
→
The scenario: Big refactor, test suite run, database migration — something that takes a while and you want to monitor from your phone.
→
How: 
claude remote-control --name "Database Migration"
Give Claude the instructions, then check in from your phone periodically. You'll see the full conversation history, tool outputs, and can approve any permission requests that come up.
Pattern 3: Quick Mobile Check-In
→
The scenario: Already in a session, suddenly need to leave — meeting, errand, etc.
→
How: 
/rc   # type this in your existing Claude Code session
Scan the QR, pocket your phone, walk to your meeting. If Claude needs approval or finishes a task, you'll see it on your phone.
💡 Permission modes + Remote Control

In Auto-Accept Edits mode, Claude keeps working without waiting for you. In Normal mode, permission prompts show up on your phone for approval. Pick the mode that matches your comfort level with the task.

Gotchas & Limitations
⚠️
One remote connection at a time. Each Claude Code session supports only one remote connection. Opening the session URL on a second device takes over from the first.
⚠️
Your terminal must stay open. Remote Control is a local process. Closing the terminal or killing the claude process ends the session. Your laptop can sleep briefly (it auto-reconnects), but shutting down the terminal kills it entirely.
⚠️
Extended outages time out. Network outages lasting ~10+ minutes will cause the session to time out. Restart with claude remote-control if that happens.
⚠️
API keys don't work. Remote Control requires authentication through claude.ai (Pro/Max/Team/Enterprise). If you're using an API key, switch to claude auth login authentication first.
⚠️
Long-lived tokens don't work either — even with the right plan. Tokens created via claude setup-token or set through the CLAUDE_CODE_OAUTH_TOKEN environment variable are inference-only — they only carry the user:inference OAuth scope, which lets you make AI requests but nothing else. Remote Control needs a full-scope token: one that includes both user:inference and user:profile. That second scope is what allows Claude Code to register the session, identify your account, and broker the remote connection through Anthropic's servers.

For example: if you set up auth using claude setup-token (common for CI pipelines or sharing tokens across machines), you'll hit this exact error when you run /remote-control: 
Remote Control requires a full-scope login token.
Long-lived tokens (from claude setup-token or CLAUDE_CODE_OAUTH_TOKEN)
are limited to inference-only for security reasons.
Run claude auth login to use Remote Control.
The fix is to run claude auth login, which opens a browser OAuth flow and grants both scopes. This is the only auth path that works with Remote Control.
💡
Everything still runs locally. Your phone isn't running code — it's just sending messages. Your machine does all actual work. Local MCP servers, custom skills, file access, and tools all work exactly as they do in the terminal.
💡
Session names help you find things. If you run multiple Remote Control sessions (different projects, terminals), use --name so you can tell them apart in the session list on claude.ai/code.
🚫 Common Mistake

Confusing Remote Control with Claude Code on the Web. They look similar but are fundamentally different — Remote Control routes to YOUR machine; the web version runs on Anthropic's cloud. Only Remote Control gives you access to your local files, MCP servers, and tools.

Remote Control vs Claude Code on the Web

These are two different things and people frequently mix them up. The key question: do you need your local environment?

Feature Comparison
Feature Remote Control Claude Code on the Web
Where code runs Your local machine Anthropic's cloud infrastructure
Local file access ✓ Yes ✗ No
Local MCP servers ✓ Yes ✗ No
Custom skills / tools ✓ Yes — full local env ✗ No
Terminal must be open ✗ Yes (limitation) ✓ No — cloud manages it
Zero local setup required ✗ No ✓ Yes
Run multiple tasks in parallel ✗ One session at a time ✓ Yes — cloud compute
Code leaves your machine ✓ No (only conversation syncs) ✗ Yes — runs on cloud
✓ Use Remote Control when...
✅You need access to your local files, database, or services.
✅You have MCP servers or custom Claude Code skills configured locally.
✅You started a task at your desk and need to monitor or continue it from your phone.
✅You want your code to stay on your machine for security or compliance reasons.
✓ Use Claude Code on the Web when...
✅The task doesn't depend on your local machine at all.
✅You want to spin up multiple tasks in parallel using cloud compute.
✅You're on a device with no Claude Code installed and need to get something done quickly.
✅You don't want to worry about keeping a terminal open.
📌 Rule of thumb

If you need your local environment, use Remote Control. If you just need Claude to do something that doesn't depend on your machine, use Claude Code on the web.
