Obsidian — What It Is

A local-first, markdown-based note-taking app. Your notes are plain .md files in a folder on your computer — no cloud lock-in, no account required, readable with any text editor forever. The superpower is bidirectional linking between notes.

Core Properties
💾
Local-first. Notes are .md files on disk. No internet required. No vendor lock-in. Open with any editor.
🔗
Bidirectional links. [[wikilinks]] connect notes. Backlinks show you everything that references a note.
🆓
Free for personal use. No feature limits. Optional paid add-ons: Sync ($4/user/month), Publish ($10/month), commercial license ($50/year).
🧩
1,000+ community plugins. Extend with calendar views, kanban boards, dataview queries, git sync, AI integrations, and more.
📱
Cross-platform. Mac, Windows, Linux, iOS, Android. Sync via Obsidian Sync, iCloud, Dropbox, Google Drive, or Git — your choice.

💡 Why Obsidian for RAG

Because it's a folder of markdown files. Claude Code can read, write, and organize them directly — no API, no database, no special integration needed. The vault IS the context window-friendly knowledge store.

Core Concepts
Vault
A folder on your computer containing all your notes. Obsidian's basic unit — you can have multiple vaults for different projects.
→ For the RAG project, your vault IS the knowledge base.
Note
A single .md (Markdown) file. Supports headers, bullets, code blocks, images, tables, and math. Created with Ctrl+N.
→ Every file in your wiki/ folder is a note.
[[Wikilinks]]
Type [[note-name]] to create a link to another note. If the note doesn't exist yet, Obsidian creates it when you click. The core of linking.
→ Example: [[ai-agents/tool-use-patterns]]
Backlinks
Automatically tracked reverse-links. If note A links to note B, note B's backlinks pane shows note A. Bidirectional by default.
→ Visible in right sidebar → Backlinks panel.
Graph View
Visual map of all notes as nodes, links as edges. Local graph shows connections around one note; global graph shows the whole vault.
→ Open with Ctrl+G. More useful as your vault grows.
Frontmatter
YAML metadata at the top of a note, between --- delimiters. Used for tags, dates, properties. Queried by the Dataview plugin.
→ tags: [ai, research] in frontmatter lets you filter notes.
Tags
Typed with #tag-name in note body, or in frontmatter. Let you organize by status or type. Nested tags work: #status/in-progress.
→ Different from wikilinks — tags are for categorization, not navigation.
Command Palette
Access every Obsidian command by name. The fastest way to do anything — search, open settings, run plugins, toggle views.
→ Open with Ctrl+P. Type anything to find it.
🔗 The Linking Mental Model

Folders silo ideas. Links reveal relationships. When you type [[some-concept]] in a note, you're building a network — not just organizing a file. Over time, clusters emerge that you didn't consciously design. That emergent structure is the value.

Interface Layout
📁
Left sidebar — File Explorer. Your vault folder tree. Click to open notes. Drag to reorganize. Right-click for file options.
📝
Center — Editor. Where you write. Toggle between source mode (raw markdown) and live preview (rendered) via the top-right icon.
🔗
Right sidebar — Backlinks + Outgoing Links. Shows all notes that link to the current note (backlinks) and all notes the current note links out to.
⚙️
Bottom-left gear — Settings. Theme, hotkeys, plugins, appearance, file behavior. Most important settings for new users: Appearance and Community Plugins.
🔍
Search bar. Full-text search across the entire vault. Supports operators like file:, tag:, path:, and regex.
First Things to Do
1
Change the theme
Settings → Appearance → Themes → Manage → pick "Minimal" to start.
2
Enable Community Plugins
Settings → Community plugins → toggle off "Restricted mode" → Browse. ⚠ Required before installing any community plugin including Web Clipper and Local Images Plus.
3
Create a home note
Press Ctrl+N, name it Home. Pin it. Link your important notes from here. This becomes your vault's entry point.
4
Try wiki-linking
In any note type [[ — Obsidian opens a search through all your notes. Select one or type a new name to create a stub note.
Editor Modes Explained
👁️
Live Preview (default). Markdown renders as you type — asterisks become bold, # becomes a heading. Most people use this.
📄
Source Mode. Raw markdown, no rendering. Useful when debugging formatting or pasting from external tools.
📖
Reading View. Fully rendered, read-only. Good for reviewing long notes or using embeds/iframes.
Switch modes via the icon in top-right of the editor pane, or via Command Palette.

Recommended Community Plugins
📎
Obsidian Web Clipper Browser Ext
Clips any webpage to your raw/ folder as markdown — one click. Configured in the browser extension, not Obsidian itself. Required for the Karpathy RAG workflow.
🖼️
Local Images Plus Community
When you open a clipped note, downloads all external images into the vault. Prevents broken images when offline. Settings → Community Plugins → Browse → "Local Images Plus"
📊
Dataview Community
Query your vault like a database. List all notes with a specific tag, sort by date, show tables. Uses SQL-like syntax in code blocks. Advanced — useful once you have 50+ notes.
📅
Calendar Community
Adds a calendar widget in the sidebar. Click any day to open/create that day's note. Good for daily note workflows.
🔀
Obsidian Git Community
Auto-commits your vault to a Git repo on a schedule. Free sync + version history via GitHub. Alternative to Obsidian Sync. Requires Git installed. Best free sync option for developers.
💡 Plugin Philosophy

Start with nothing. Get comfortable with the core app for a week before adding plugins. Each plugin adds settings to learn. The best vault is the one you actually use.

Common New User Gotchas
⚠️
Community plugins need Restricted Mode off. Settings → Community plugins → toggle off "Restricted mode" before browsing. Easy to miss, confusing when you can't find the Browse button.
⚠️
[[Wikilinks]] are case-sensitive on some systems. [[My Note]] and [[my note]] may resolve to different files on Linux/Windows. Use lowercase filenames consistently.
⚠️
No built-in sync (free tier). Obsidian Sync is paid. Alternatives: iCloud (Mac/iOS), Dropbox, Google Drive, or Obsidian Git. Pick one before using on multiple devices.
⚠️
Renaming files breaks unupdated links. Use Obsidian's rename (F2 or right-click) instead of renaming via Finder/Explorer — Obsidian auto-updates all wikilinks. Direct filesystem renames break links.
⚠️
The graph looks cool but isn't how you work. Don't spend time trying to make the graph look good. Links are for navigation and discovery, not aesthetics. The graph is a byproduct, not a goal.
⚠️
Plugin overload is real. Installing 20 plugins before you understand the basics leads to settings confusion and conflicts. Start minimal, add one plugin at a time when you have a specific need.
⚠️
Don't obsess over organization before you have content. The most common trap: spending days perfecting folder structure with 5 notes. Write first. Organize later. Links matter more than folders.
Good Habits to Build Early
✅
Use lowercase-hyphenated filenames. ai-agent-overview.md not AI Agent Overview.md. Avoids cross-platform link issues.
✅
Link generously. Even to notes that don't exist yet. Stub notes mature into full articles over time. The connection matters now.
✅
Always rename via Obsidian. Right-click → Rename, or press F2. Never through Finder or File Explorer.
✅
Set up backup/sync early. Configure Git or Dropbox before you have important notes. Don't learn this lesson the hard way.

CLAUDE.md as Persistent Memory
🧠
CLAUDE.md is the "frontal cortex." The vault is the whole brain — all your notes and context. CLAUDE.md is the distilled thinking template: conventions, preferences, how you work. Claude always reads this first.
📈
It's a living document — update it over time. Periodically tell Claude: "Take a look at all our notes. Compare to our CLAUDE.md. Make them match and improve the conventions." Your assistant gets closer to Jarvis with every pass.
⚡
Use /init to bootstrap it. Run /init inside your vault project and Claude Code reads the entire structure, figures out what you're doing and why, and writes the CLAUDE.md for you automatically.
📍
Put the vault where your Claude Code projects live. Easy to navigate to in the terminal. Co-location means less friction starting sessions.
What to put in CLAUDE.md for a personal assistant vault
Tell Claude: your Obsidian conventions (all markdown files must follow Obsidian format, use [[wikilinks]], lowercase-hyphenated filenames), your folder structure and what each folder is for, how you want daily notes formatted, your preferred writing style, and any recurring context about you and your projects that Claude should always have available.
The Anti-CLAUDE.md Study — And Why It Doesn't Apply Here
📄
A published study found CLAUDE.md hurts coding agents. "Evaluating Agents: Are Repository-Level Context Files Helpful for Coding Agents?" concluded that convention files are often a net negative — they enforce rules that conflict with how real projects are structured across UI, backend, auth, etc.
✅
But this does NOT apply to personal assistant vaults. The study is about code conventions clashing with project architecture. A personal assistant vault's conventions are about how you think and how to write markdown files — not code. Those conventions are always valid and always welcome context.
⚠️
Avoid CLAUDE.md for coding projects. If you're using Claude Code on an actual software codebase, the study's warning applies. CLAUDE.md conventions about auth patterns or file structure can actively hurt performance there. Keep coding projects separate from your knowledge vault.
📈 The Compounding Effect

Daily notes for two weeks: manageable. Daily notes for two years: a rich context map of how you think and what you've worked on. The personal assistant use case is where breadth compounds over time — wide topics, long time horizons. The vault gets more useful the longer you use it. Start now, even if it's small.

Quick Start: Claude Code Inside Your Vault
1
Put your vault where your projects live
Create or move the vault to wherever you normally cd in the terminal. Makes starting sessions frictionless.
2
Open terminal in vault root → run claude
You're now inside a Claude Code project where every file is a markdown note. Claude can read, write, and link them all.
3
Run /init to auto-generate CLAUDE.md
Claude scans the vault, understands the structure, and writes the conventions file for you. Edit it to add personal context. 💡 Or write it manually — see the CLAUDE.md tab in the RAG reference tool for a full template.
4
Tell Claude to follow Obsidian conventions
Add to CLAUDE.md: "All markdown files must follow Obsidian conventions — use [[wikilinks]], lowercase-hyphenated filenames, and update relevant index files after any changes." ✓ Claude Code already knows Obsidian conventions — you just need to activate them.
5
Start using it as a personal assistant
Brain-dump, ask questions, request research. Claude Code writes structured notes into the vault. You read them in Obsidian. The graph fills in. The flywheel starts.
