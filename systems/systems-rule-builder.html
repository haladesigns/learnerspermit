Usage guide
Quick start ▾
1
Drag a primitive from the toolbar onto the canvas — State, Input, Transition, etc.
2
Click a node to select it and edit its label, description, and type-specific fields in the Properties panel.
3
Connect nodes by pressing C or clicking → Connect, then clicking source → target. Or drag the bottom port dot of any node.
4
Export your graph as Text rules, JSON, or a ready-to-paste LLM prompt via the Export button.
Tip: Build outward from the boundary. Place a Boundary node first to define your system scope, then populate states and transitions inside it.
Primitives ▾
S
State
A stable condition the system occupies. Examples: Idle, Processing, Error, Locked. Every system needs at least one state.
T
Transition
A guarded change from one state to another. Has a when condition and a then action. Wire: State → Transition → State.
I
Input
External data, signals, or energy entering the system boundary. Examples: HTTP request, sensor reading, user action.
O
Output
Results, responses, or signals leaving the system. Examples: API response, log entry, actuator command.
F
Feedback
An output that loops back as input. Set polarity: negative (stabilising — thermostat) or positive (amplifying — viral growth).
B
Boundary
Defines what is inside vs outside your system. Use it to label the overall system or a major scope line. Renders with a dashed border.
S
Subsystem
A nested, self-contained component inside the main system. Use to group related states when a section is complex enough to stand alone. Renders with a double border.
Canvas interactions ▾
⊕
Add node
Drag any chip from the toolbar and drop it anywhere on the canvas.
↖
Select node
Click a node to select it and open its properties in the sidebar.
⤢
Move node
Click and drag any node to reposition it on the canvas.
✋
Pan canvas
Click and drag on empty canvas space to pan the viewport.
⊙
Connect nodes
Drag the bottom port dot of a node to another node's top port. Or press C to enter Connect mode, then click source → target.
✕
Delete node
Hover a node and click the ✕ badge, or select it and press Del.
Multi-select ▾
⇧
Shift+click
Toggle individual nodes in or out of the multi-selection. Highlighted with a purple ring.
⬚
Shift+drag (canvas)
Draw a marquee box on empty space. All nodes overlapping the box are selected on release.
⤢
Group move
With multiple nodes selected, drag any highlighted node — the whole group moves together preserving relative positions.
⌫
Group delete
Press Del or click the Delete button in the sidebar to remove all selected nodes and their connections at once.
Tip: Click empty canvas or press Esc to clear multi-selection.

Export formats ▾
JSON
Machine-readable. Best for re-importing, programmatic processing, or embedding in a system prompt as structured data. Includes full node metadata, connection list, and a transition summary table.
TEXT RULES
Human-readable structured text. Sections for each primitive type, a connections list, and a transition table with guards and actions. Good for pasting directly into an LLM context window.
LLM PROMPT
A complete, ready-to-send prompt wrapping the text rule graph. Asks the LLM to summarise the system, identify feedback loops, describe failure modes, and analyse boundary conditions.
Why structured output? A graph of states and transitions gives an LLM explicit causal relationships, reducing hallucination compared to prose descriptions. The structured format also compresses token usage significantly.
Import ▾
Supported files
.json JSON exports from this tool.
.txt .md Text rule exports, or LLM prompt exports (the rule graph is automatically extracted from between the --- RULE GRAPH --- markers).
Merge vs replace
The import dialog lets you choose: clear canvas first (default) replaces everything, or uncheck to merge the imported graph alongside existing nodes. IDs are remapped to prevent collisions.
