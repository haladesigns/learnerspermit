End-to-End Flow

One prompt triggers this entire chain. Each stage is handled by a different system — Claude Code is only the orchestrator.

Pipeline Stages
Stage 1 — Query
User writes a single natural-language prompt. Example: "Search YouTube for claude code skills, push to NotebookLM, get analysis, produce a blueprint-style infographic."
Stage 2 — YouTube Search (yt-search skill)
Claude Code invokes the yt-search skill which runs search.py via yt-dlp. Returns up to N video URLs with title, channel, views, duration, engagement ratio. No YouTube API key required.
Stage 3 — Notebook Creation (notebooklm-py)
Claude Code calls the NotebookLM skill which uses notebooklm-py to create a new notebook and push all video URLs as sources. NotebookLM ingests captions automatically. Limit: 50 sources per notebook.
Stage 4 — Offloaded Analysis (Google / Gemini)
Claude Code sends a query to NotebookLM via the API. NotebookLM runs cross-source analysis grounded in the actual video captions — citations included. Zero Claude tokens spent here. Google's Gemini does the work.
Stage 5 — Deliverable Generation
Claude Code requests a deliverable (infographic, slide deck, audio overview, flashcards, mind map). NotebookLM generates it. Claude Code retrieves the output and saves it to the project folder.

5-Minute Setup

Two components to install: the YouTube search skill and the notebooklm-py unofficial API. Both are independent — install in any order.

Part A — YouTube Search Skill
1
Install the yt-dlp dependency
Open a terminal (not Claude Code) and run: 
pip install yt-dlp
2
Get the skill — two options
Option A: Tell Claude Code to build it. Prompt: "Build me a custom skill using yt-dlp that can search YouTube and return structured video metadata."
Option B: Download the pre-built YT-search-skill.md setup file from Chase's free School community, then hand it directly to Claude Code. 💡 Option B is faster and guaranteed to match the walkthrough exactly.
3
Install the skill inside Claude Code
Either run the install command in terminal or tell Claude Code:
"Install this skill from the MD file I'm providing." 💡 Skills are just plain-text prompt files — no compiled code, no dependencies of their own.
Part B — NotebookLM API (notebooklm-py)
1
Clone / install notebooklm-py
Open a separate terminal (not the Claude Code window) and paste the install commands from the GitHub README: 
# from https://github.com/teng-lin/notebooklm-py
pip install notebooklm-py   # or follow repo instructions
⚠ Run this in a normal terminal, not inside Claude Code.
2
Authenticate with Google
Still in the separate terminal, run the one-time login command: 
notebooklm login
A Chrome window opens. Sign into your Google account. You only do this once — credentials are cached locally. 💡 No API key required. Authentication is handled via browser OAuth flow.
3
Install the NotebookLM skill inside Claude Code
Run the skill install command from the repo (or tell Claude Code to install it). The SKILL.md file teaches Claude Code the correct CLI calls, notebook creation syntax, and how to request deliverables. 💡 Remember: the skill is separate from the program. The program is the Python API; the skill is the prompt file that tells Claude Code how to use it.
⚡ Quick Verify

Type /yt-search claude code tutorial in Claude Code. If you get back structured results (title, views, URL), the YouTube skill is working. Then ask Claude Code to create a test notebook in NotebookLM — if it appears in your NotebookLM dashboard, you're fully connected.

③ How To Use — The Command Spec

This section tells Claude Code the exact bash command to run, with full path, required args, and optional flags. Claude treats this literally — it copies the structure when constructing the actual call.

## How to use

Run the search script:

python "C:/Users/Chase/.claude/skills/yt-search/scripts/search.py" <query> [--count N] [--months N] [--no-date-filter]

### Parameters
- <query>     — search terms (required)
- --count N   — number of results (default: 20)
- --months N  — only last N months (default: 6)
- --no-date-filter — include videos of any age
⚠ Hardcoded Path

The script path is absolute and user-specific: C:/Users/Chase/.... When adapting this skill, update the path to match your own system. On macOS/Linux this would be something like /Users/yourname/.claude/skills/yt-search/scripts/search.py.

④ Examples — Few-Shot Prompting for Claude

The examples section is few-shot prompting embedded in the skill file. Claude Code uses these to understand the range of valid invocations — not just the basic case, but edge cases like filtering by recency or disabling date filters entirely.

⑥ Notes — Implementation Details
📦
Dependency: yt-dlp must be installed (pip install yt-dlp). No YouTube API key needed.
2×
Oversample buffer: The script fetches 2× the requested count to compensate for date-filter dropouts. If you ask for 20, it fetches 40 internally, then trims.
📡
Subscriber counts: Come from yt-dlp metadata — may occasionally return N/A if unavailable.
🔀
stderr vs stdout: Status messages (searching, filtering) go to stderr; formatted results go to stdout. This keeps the result clean for Claude to parse.

# YouTube Search

Search YouTube using yt-dlp and return structured, formatted results
with engagement metrics.

## How to use

Run the search script:

python "C:/Users/Chase/.claude/skills/yt-search/scripts/search.py" <query> [--count N] [--months N] [--no-date-filter]

### Parameters
- `<query>`        — search terms (required)
- `--count N`      — number of results to return (default: 20)
- `--months N`     — only show videos from the last N months (default: 6)
- `--no-date-filter` — include videos of any age

### Examples

# Basic search (top 20, last 6 months)
python "C:/Users/Chase/.claude/skills/yt-search/scripts/search.py" claude code tutorial

# Fewer results, shorter window
python "C:/Users/Chase/.claude/skills/yt-search/scripts/search.py" obsidian workflow --count 5 --months 3

# All time, no date filter
python "C:/Users/Chase/.claude/skills/yt-search/scripts/search.py" AI agents --count 10 --no-date-filter

## Output format

Each result includes:
- Title
- Channel name and subscriber count
- View count
- Duration
- Upload date
- Engagement ratio (views / subscribers) — higher means the video
  outperformed the channel's base
- Direct URL

Results are separated by dividers for easy scanning. Status messages
(searching, filtering) go to stderr; the formatted results go to stdout.

## Notes

- yt-dlp must be installed (`pip install yt-dlp`)
- The script fetches 2x the requested count to compensate for date
  filtering
- Subscriber counts come from yt-dlp metadata and may occasionally
  be unavailable (shown as N/A)
🧠 Skill Design Takeaway

A skill file has five jobs: (1) tell Claude when to fire via the description field, (2) give the exact command syntax, (3) show examples for edge cases, (4) describe expected output so Claude can parse it, (5) document gotchas in Notes. Every great skill covers all five. The frontmatter description is the most critical — bad trigger phrases mean the skill never gets used.

✓ Do This
✅Open a separate terminal for pip installs and notebooklm login — not inside Claude Code.
✅Authenticate with notebooklm login before trying any API calls.
✅Preview your sources first with /yt-search alone before pushing to NotebookLM.
✅Update the hardcoded script path in the skill file to match your system.
✅Use --no-date-filter for evergreen or historical topics.

✗ Avoid This
❌Don't run install commands inside Claude Code — use a real terminal.
❌Don't confuse the skill (prompt file) with the program (Python API). Both are required.
❌Don't push 50 sources at once without checking quality first — engagement ratio helps filter.
❌Don't expect real-time data — yt-dlp metadata (especially subscriber counts) can be slightly stale.
❌Don't rely on Claude Code to do the analysis — that defeats the purpose. Let NotebookLM handle it.
Key Warnings
⚠️
Unofficial API: notebooklm-py reverse-engineers NotebookLM's internal endpoints. Google could change these without notice and break the integration. Check the GitHub repo for updates if things stop working.
⚠️
Path is user-specific: The skill file hardcodes C:/Users/Chase/.... You must update this to your own path or the script call will always fail.
⚠️
50-source limit: NotebookLM caps notebooks at 50 sources. The script fetches 2× your requested count internally to compensate for date filtering — keep final push count under 50.
⚠️
Login is one-time but session-bound: Credentials are cached locally after the first login. If your session expires, run notebooklm login again.
