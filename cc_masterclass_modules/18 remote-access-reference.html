3 Ways to Access Claude Code Remotely

Anthropic dropped three remote-access features in quick succession. They sound similar but solve different problems. For 99% of people, Remote Control is all you need. Here's the full breakdown.

Remote Control
📱
Best for most people. Control an active Claude Code terminal session from your phone via the official Claude app.
🔴
Live stream of terminal output — you see everything in real time, not just a completion message.
✋
Can approve or deny permissions from your phone (unless running --dangerously-skip-permissions).
⚡
Easiest setup. Uses outbound HTTPS polling + streaming TLS. Auto-reconnects after a 10-minute timeout.
🌐
Available on iOS, Android, and claude.ai/code — not just the mobile app.
Dispatch
🖥️
Claude Cowork (desktop app) only. Not for Claude Code / terminal. Different product entirely.
🔥
Fire-and-forget. Send tasks from your phone; Cowork executes them in a sandboxed desktop environment.
🔗
Access to 38+ native integrations — calendar, email, files, etc. Great for non-technical tasks.
📱
Pairs phone to desktop via QR code. Runs tasks inside an Electron + sandboxed environment.
🍎
Mac only. Requires the Claude Cowork desktop app — not available on Windows or Linux.
Channels
💬
Talk to Claude Code via Telegram or Discord. Text a task, go grab coffee — async by design.
⚙️
Runs via MCP server. Same protocol as the rest of Claude Code's ecosystem. More hackable.
🛠️
More developer-friendly — custom channels buildable in ~30 lines of TypeScript. Good for tinkerers.
⚠️
Only returns completion summaries, not a live terminal stream. No mid-task permission prompts.
⚠️ Key Distinction

Remote Control and Channels are both for Claude Code (terminal). Dispatch is for Claude Cowork (desktop app). They are not interchangeable.

💡 The Big Shift

Channels represents a move from synchronous "ask-and-wait" to asynchronous "text-and-forget" workflows. Remote Control keeps the synchronous model, just on your phone.

Which One Do You Need?
📱 Remote Control
→You have a Claude Code terminal session running and want phone access to it.
→You want to see real-time output or approve tool calls remotely.
→You hate work-life balance but love simplicity.
→You don't want to set up a bot or MCP server.
🖥️ Dispatch
→You're on a Mac using Claude Cowork (the desktop app).
→You want to fire off non-coding tasks: emails, calendar, documents.
→You don't need a terminal — you want the full desktop app scope.
→You don't need to watch it work, just get results.
💬 Channels
→You want Claude Code available as a Telegram or Discord bot.
→You want async workflows — queue tasks and check results later.
→You want to build custom integrations via MCP.
→You like tinkering under the hood.
⚠️ All Three: Research Preview

Syntax and behavior may change for all three features. Don't hardcode Channels plugin commands into scripts you depend on — check the docs when things break.

📌 Before You Start

Channels needs an active Claude Code session open on your computer. It is a local process, not a cloud service. Close the terminal = no Channels.

Telegram Setup
1
Create a bot via BotFather
Open Telegram → message @BotFather → /newbot. Follow prompts. Save the token it gives you.
2
Install the Telegram plugin 
/plugin install telegram@claude-plugins-official
3
Configure with your token 
/telegram:configure <token>
4
Restart Claude Code with Channels enabled 
claude --channels plugin:telegram@claude-plugins-official
5
DM your bot on Telegram
Send any message to get your pairing code.
6
Pair the session 
/telegram:access pair <code>
7
Lock down access 
/telegram:access policy allowlist
⚠ Do this — otherwise anyone who finds your bot could use it.
Discord Setup
1
Create app in Discord Developer Portal
Go to discord.com/developers → New Application → Bot tab → enable Message Content Intent.
2
Invite bot with required permissions
OAuth2 → URL Generator. Scopes: bot. Permissions needed: 
View Channels
Send Messages
Read Message History
Attach Files
Add Reactions
3
Install the Discord plugin 
/plugin install discord@claude-plugins-official
4
Configure with your bot token 
/discord:configure <token>
5
Restart Claude Code with Channels enabled 
claude --channels plugin:discord@claude-plugins-official
6
DM the bot → get pairing code → pair 
/discord:access pair <code>
7
Lock down access 
/discord:access policy allowlist
⚠ Do this — same reason as Telegram.
💡 Setup Tip

For Telegram: the official docs describe this as four steps — copy the setup prompt into Claude Code and it'll walk you through it interactively. Easier than doing it manually.

The Catch

Channels only works while your Claude Code session is open. It is a local process, not a cloud service. No session = no Channels. Remote Control handles disconnects better — it auto-reconnects after a 10-minute timeout — but still requires the terminal session to be running. Here are strategies to keep Channels alive.

Keep-Alive Strategies
1
tmux + LaunchAgent (macOS)
Run Claude Code inside a persistent tmux session. Use launchctl (via a .plist LaunchAgent) to automatically restart the session daily or on login. This is the most robust option. 
# Create tmux session on startup via LaunchAgent
tmux new-session -d -s claude 'claude --channels plugin:telegram@claude-plugins-official'
💡 Tip: Use tmux attach -t claude to check in from a local terminal.
2
Ralph Plugin — Autonomous Loops
Use the Ralph plugin's /ralph-loop command to keep the session active with autonomous task loops. 
/ralph-loop # keeps session alive via periodic activity
💡 Good if you're already using Ralph for autonomous work.
3
Custom Heartbeat Channel
Build a simple MCP channel (~30 lines of TypeScript) that sends periodic pings to keep the session from timing out. The MCP architecture makes this straightforward. 
// Minimal heartbeat MCP channel concept
setInterval(() => {
  channel.ping(); // keep-alive every N minutes
}, 5 * 60 * 1000);
💡 Most developer-friendly option — you control the timing and logic.
Gotchas
⚠️
No session = silent failure. If your computer sleeps, restarts, or Claude Code crashes, Channels goes dark. Your Telegram/Discord messages won't error — they just won't get processed.
⚠️
No permission prompts via Channels. If Claude hits something it needs approval for mid-task, it can't ask you through Telegram. Plan your tasks accordingly or pre-configure permissions.
⚠️
Research Preview syntax changes. Plugin install commands and configure flags may change. Don't hardcode these in long-lived scripts without checking the docs periodically.
⚠️
Access policy matters. If you skip /telegram:access policy allowlist, anyone who finds your bot ID could potentially send it tasks. Always lock it down.
💡
Custom channels are buildable. The MCP-based architecture means you can build your own channel in ~30 lines of TypeScript — not just Telegram/Discord. SMS, Slack, webhook endpoints, etc. are all possible.
