Godot 4 — Every 2D Node

All 2D nodes inherit from Node2D, which tracks position, rotation, and scale. Click any node card to see what it does, its scene tree structure, and the GDScript code that brings it to life. Nodes are ingredients — scenes are recipes built from them.

Node2D — The Root of Everything 2D
📐
position — Vector2. Where the node is in 2D space relative to its parent.
🔄
rotation — float in radians. rotation_degrees is the same property in degrees.
🔍
scale — Vector2. Default is Vector2(1, 1). Half scale = Vector2(0.5, 0.5).
🌍
global_position — position in world space, not relative to parent. Use this for distance checks between nodes.
👁️
visible — bool. hide() / show() are shorthand methods.
Node Hierarchy Map — Physics Bodies
CollisionObject2D (base — can't use standalone) └─ PhysicsBody2D (base — can't use standalone) └─ StaticBody2D walls, floors — doesn't move └─ AnimatableBody2D moving platforms — pushes others └─ RigidBody2D physics objects — gravity, forces └─ CharacterBody2D ⭐ player / enemies — code-controlled └─ Area2D detection zones, custom physics regions
The Golden Rule: Physics Node Choice
🎮
Player / Enemy → CharacterBody2D — precise code control
🧱
Walls / Ground → StaticBody2D — never moves
🪨
Physics Objects → RigidBody2D — gravity, bounce, forces
⚙️
Moving Platform → AnimatableBody2D — scripted/animated, pushes bodies
🔍
Pickup / Trigger Zone → Area2D — no physics, just detection
CollisionShape2D — Required by All Physics
📦
Every physics node needs a CollisionShape2D (or CollisionPolygon2D) as a child. This defines the physical shape.
⭕
Common shapes: CircleShape2D, RectangleShape2D, CapsuleShape2D. Set in the Inspector.
⚠️
Forgetting CollisionShape2D is the #1 beginner mistake — physics body exists but nothing can collide with it.
💡
Use CollisionPolygon2D for irregular shapes — draw the polygon directly in the editor.
📐 Godot's Coordinate System

Y-axis points down — positive Y moves a node downward. Vector2(0, -1) is up. Angles increase clockwise. This differs from standard math but matches screen coordinates.

🎮
CharacterBody2D
PHYSICS
Specialized physics body for player/enemy control. Has move_and_slide() and move_and_collide(). Not affected by gravity automatically — you apply it in code.
🧱
StaticBody2D
PHYSICS
Physics body that cannot be moved. Other bodies collide with and slide along it. Use for walls, floors, ceilings. Moving it in code teleports, does not push.
🪨
RigidBody2D
PHYSICS
Full physics simulation — gravity, forces, bounce, friction. Use for crates, balls, debris. Not for player characters. Control via apply_force(), apply_impulse().
⚙️
AnimatableBody2D
PHYSICS
Like StaticBody2D but can move via code or animation. Not affected by gravity. When moved, pushes other physics bodies — perfect for moving platforms and doors.
🔗
Joint2D Nodes
PHYSICS
Connects two physics bodies together. Three types: DampedSpring (spring), GrooveJoint (piston / slide axis), PinJoint (pivot point — free rotation).
⚠️ Collision Layers & Masks

Every physics body and Area2D has a collision_layer (what layer this object is on) and collision_mask (what layers it detects). Two objects only interact if one's layer is in the other's mask. This is how you prevent enemies from colliding with each other while still colliding with the player.

🖼️
Sprite2D
VISUAL
Renders a static texture to the screen. Supports region rendering from a texture atlas. Set flip_h / flip_v to mirror the sprite.
🎞️
AnimatedSprite2D
VISUAL
Renders animated sprites from a SpriteFrames resource. Built-in animation editor for setting up sprite sheet animations. Has play(), stop(), animation_finished signal.
📷
Camera2D
VISUAL
Sets the viewport's view point. Usually a child of the player. Has drag, smoothing, and zoom settings. Limit properties clamp camera to world bounds.
📏
Line2D
VISUAL
Draws a 2D polyline through a set of points. Configurable width, taper, corner rounding, and texture. Useful for ropes, trails, debug paths.
⬡
Polygon2D
VISUAL
Renders a filled polygon. Points are set in an array or edited visually in the editor. Supports UV mapping for textures.
🗺️
TileMap
VISUAL
The main node for building 2D tile-based levels. Paint tiles in the editor. Supports physics layers, navigation layers, and auto-tiling. One of the largest nodes in Godot.
🌄
ParallaxBackground + ParallaxLayer
VISUAL
Scrolling background layers at different speeds. ParallaxBackground is the container; each ParallaxLayer holds sprites and has a motion_scale for parallax depth.
📍
Marker2D
UTILITY
Debug-only node — invisible in builds. Shows position/rotation in the editor. Use as spawn points, waypoints, bullet origins, or any invisible reference position.
🔍
Area2D
DETECTION
Two roles: 1) Detection field — emits signals when bodies enter/exit. 2) Physics override zone — local gravity, wind, audio. Use for pickups, damage zones, water.
📡
RayCast2D
DETECTION
Casts an infinitely thin ray in a direction, reports first collision. Use for line-of-sight checks, ground detection, laser beams.
🔎
ShapeCast2D
DETECTION
Like RayCast2D but uses a shape (rectangle, circle) instead of a line. Better for detecting whether a character can fit through a gap, or area-of-effect checks.
👁️
VisibleOnScreenNotifier2D
DETECTION
Emits screen_entered / screen_exited signals. VisibleOnScreenEnabler2D variant auto-enables/disables its parent node — perfect for performance optimisation.
🔀
RemoteTransform2D
UTILITY
Copies its own transform to a remote node without being its parent. Selective — can sync only position, only rotation, or all. Useful for camera rigs and UI that follow world objects.
🛤️
Path2D + PathFollow2D
UTILITY
Path2D defines a Bézier curve. PathFollow2D binds a node to that curve and moves it along it. Use for patrol routes, cutscene movement, conveyor belts.
🔊
AudioStreamPlayer2D
AUDIO
Plays audio from a position in 2D space. Volume attenuates with distance. Settings: volume, pitch, max_distance, attenuation. Needs an AudioStream resource.
👂
AudioListener2D
AUDIO
Overrides the camera as the listening point for 2D audio. By default, the camera is the listener. Add this if you want audio to be heard from a different position.
✨
GPUParticles2D
VISUAL
GPU-accelerated 2D particle system. Faster than CPUParticles2D for large counts. Controlled by a ParticleProcessMaterial. Best learned by adding one and experimenting.
💨
CPUParticles2D
VISUAL
CPU-side particle system. More portable than GPU particles (works on all hardware). Slower for very large particle counts. Same visual results for small-to-medium effects.
AudioStreamPlayer2D — GDScript
extends Node2D

@onready var footstep_sfx: AudioStreamPlayer2D = $FootstepSFX
@onready var hit_sfx:      AudioStreamPlayer2D = $HitSFX

func play_footstep() -> void:
    # Randomise pitch slightly so repeated sounds feel natural
    footstep_sfx.pitch_scale = randf_range(0.9, 1.1)
    footstep_sfx.play()

func play_hit() -> void:
    # Stop whatever was playing and restart
    hit_sfx.stop()
    hit_sfx.play()

# Wait for a sound to finish before doing something
func play_then_die() -> void:
    hit_sfx.play()
    await hit_sfx.finished
    queue_free()
🌑
CanvasModulate
LIGHTING
Required to enable 2D shadows. Darkens the scene to a base colour so 2D lights can illuminate it. Add one to the scene root before adding any 2D lights.
💡
PointLight2D
LIGHTING
Lights the scene using a texture, not a point. The shadow is cast from the node's center. Give it a circular texture for a torch/lamp effect. Needs CanvasModulate in scene.
☀️
DirectionalLight2D
LIGHTING
Illuminates the entire scene from one angle — like the sun or moon. Shadows don't warp with distance (unlike PointLight2D). Set the rotation to control light angle.
🌒
LightOccluder2D
LIGHTING
Defines the shadow-casting shape of an object. Add as a child of any node that should cast shadows. Draw its OccluderPolygon2D to match the object's silhouette.
🖥️
BackBufferCopy
FX
Captures a region of the screen (or all of it) into a texture. The texture is accessible in shaders via the hint_screen_texture variable. Use for glass, portals, distortion shaders.
🎨
CanvasGroup
FX
Merges all child nodes into a single draw call. Makes translucent children blend correctly as a unit (e.g. a bottle and its liquid). Useful for compositing effects.
2D Lighting — Minimal Setup
Node2D (scene root) ├─ CanvasModulate color: #1a1a2e ← darkness base ├─ TileMap (your level) │ └─ LightOccluder2D (on walls that cast shadow) └─ Player └─ PointLight2D texture: torch.png ← follows player 
# Flicker a torch light with noise
extends PointLight2D

@export var base_energy: float = 1.0
@export var flicker_amount: float = 0.15
var time := 0.0

func _process(delta: float) -> void:
    time += delta
    # sin wave at different frequencies = organic flicker
    energy = base_energy + sin(time * 7.3) * flicker_amount \
                         + sin(time * 13.1) * flicker_amount * 0.5
🦴
Skeleton2D + Bone2D
ANIMATION
2D skeletal animation system. Skeleton2D is the root; Bone2D nodes are the skeleton bones. Skin a Polygon2D mesh to it. Third-party tools like DragonBones/Spine can also export for Godot.
🗺️
NavigationRegion2D
NAVIGATION
Defines a walkable area for AI pathfinding. Unlike 3D, you must draw the region manually (no auto-bake). Pair with NavigationAgent2D on the enemy to find paths through it.
⛔
NavigationObstacle2D
NAVIGATION
A dynamic obstacle in the navigation mesh. Moving characters or objects the AI should avoid. Works in real-time unlike static NavRegion baking.
🧩
MeshInstance2D
VISUAL
Renders a 3D mesh in 2D space. Useful for rendering 3D objects flat on a 2D scene, or for custom procedural geometry. Rare in pure 2D games.
🌿
MultiMeshInstance2D
VISUAL
Renders many copies of one mesh efficiently. GPU-instanced, so rendering 10,000 grass blades is far cheaper than 10,000 Sprite2D nodes. Great for environmental density.
Path2D + PathFollow2D — Patrol Route
Node2D ├─ Path2D draw patrol curve in editor └─ PathFollow2D attaches enemy to the path └─ Enemy CharacterBody2D + visuals 
# Attach this script to PathFollow2D
extends PathFollow2D

@export var speed: float = 80.0    # pixels per second along path
var direction: float = 1.0          # 1 = forward, -1 = reverse

func _process(delta: float) -> void:
    progress_ratio += speed * direction * delta  # 0.0 → 1.0 = full path

    # Reverse direction at each end
    if progress_ratio >= 1.0:
        progress_ratio = 1.0
        direction = -1.0
    elif progress_ratio <= 0.0:
        progress_ratio = 0.0
        direction = 1.0

Real Game Scenarios

These are complete, working scene + script combos — the kind of thing you'll build in your first games. Each shows how nodes work together with GDScript.

Scenario 1 — Top-Down Player Movement
📐 Scene Tree
Player (CharacterBody2D) ├─ Sprite2D your character texture ├─ CollisionShape2D CapsuleShape2D └─ Camera2D follows the player automatically 
# player.gd — attach to CharacterBody2D
extends CharacterBody2D

const SPEED: float = 200.0

func _physics_process(delta: float) -> void:
    # get_vector returns a normalised Vector2 from 4 input actions
    # ui_left/right/up/down are built-in — no Input Map setup needed
    var dir := Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down")

    # Set the built-in velocity property — move_and_slide() reads it
    velocity = dir * SPEED

    # Flip sprite to face direction of movement
    if dir.x != 0:
        $Sprite2D.flip_h = dir.x < 0

    # Handles collision response, slides along walls
    move_and_slide()
💡 Key Points

Use _physics_process, not _process, for all movement — ensures consistent physics at 60Hz. get_vector() normalises diagonals so you don't move faster at 45°. velocity is a built-in property on CharacterBody2D — set it, then call move_and_slide().

Scenario 2 — Platformer with Jump & Gravity
📐 Scene Tree
Player (CharacterBody2D) motion_mode = Grounded ├─ AnimatedSprite2D animations: idle, run, jump └─ CollisionShape2D CapsuleShape2D 
extends CharacterBody2D

const SPEED:       float = 200.0
const JUMP_FORCE:  float = -400.0  # negative = upward (Y-down)
const GRAVITY:     float = 900.0

@onready var anim: AnimatedSprite2D = $AnimatedSprite2D

func _physics_process(delta: float) -> void:
    # Apply gravity when airborne
    if not is_on_floor():
        velocity.y += GRAVITY * delta

    # Jump — only from the ground
    if Input.is_action_just_pressed("ui_accept") and is_on_floor():
        velocity.y = JUMP_FORCE

    # Horizontal movement
    var h := Input.get_axis("ui_left", "ui_right")
    velocity.x = h * SPEED

    # Update animation
    if not is_on_floor():
        anim.play("jump")
    elif h != 0:
        anim.play("run")
        anim.flip_h = h < 0
    else:
        anim.play("idle")

    move_and_slide()
💡 Key Points

is_on_floor() returns true when CharacterBody2D's motion_mode is Grounded and it's touching a floor. is_action_just_pressed() fires once per press — use this for jump so holding down doesn't multi-jump. Y-axis is inverted: negative = up.

Scenario 3 — Coin Pickup (Area2D)
📐 Scene Tree
Coin (Area2D) ├─ AnimatedSprite2D spin animation └─ CollisionShape2D CircleShape2D — slightly larger than sprite 
# coin.gd — attach to Area2D
extends Area2D

signal collected(value: int)   # tell the world this coin was picked up

@export var coin_value: int = 10

func _ready() -> void:
    # Connect the Area2D built-in signal to our handler
    body_entered.connect(_on_body_entered)
    $AnimatedSprite2D.play("spin")

func _on_body_entered(body: Node2D) -> void:
    # Only react to the player, not other bodies
    if body.is_in_group("player"):
        collected.emit(coin_value)     # parent scene listens to this
        queue_free()                    # remove coin from scene

# In the level scene, listen for coins:
# $Coin.collected.connect(_on_coin_collected)
# func _on_coin_collected(value: int) -> void:
#     score += value
💡 Key Points

Area2D's body_entered signal fires when any physics body enters. Filter by group (is_in_group) to react only to the player. Emit a custom signal upward so the coin doesn't need to know about the score system — clean separation of concerns. queue_free() safely removes a node at the end of the frame.

Scenario 4 — Enemy Line-of-Sight (RayCast2D)
📐 Scene Tree
Enemy (CharacterBody2D) ├─ Sprite2D ├─ CollisionShape2D └─ SightRay (RayCast2D) target_position: Vector2(200, 0) 
extends CharacterBody2D

enum State { PATROL, CHASE }
var state: State = State.PATROL

@export var patrol_speed: float = 60.0
@export var chase_speed:  float = 120.0

@onready var ray: RayCast2D = $SightRay

var player: Node2D = null

func _physics_process(delta: float) -> void:
    # Point ray toward the player each frame
    if player:
        ray.target_position = ray.to_local(player.global_position)

    # Check what the ray is hitting
    if ray.is_colliding():
        var hit := ray.get_collider()
        if hit.is_in_group("player"):
            state = State.CHASE
            player = hit
        else:
            state = State.PATROL  # wall blocking view
    else:
        state = State.PATROL

    # Act on state
    match state:
        State.PATROL:
            velocity.x = patrol_speed  # simple walk right (add reversal logic)
        State.CHASE:
            var dir := (player.global_position - global_position).normalized()
            velocity = dir * chase_speed

    move_and_slide()
💡 Key Points

RayCast2D's target_position is in local space — use to_local() to convert the player's world position. is_colliding() only returns true on the frame after the ray updates; get_collider() returns the first thing hit. This pattern gives you line-of-sight that respects walls correctly.

Scenario 5 — AnimatedSprite2D Full Integration
# Handles animations on a CharacterBody2D player
extends CharacterBody2D

@onready var anim: AnimatedSprite2D = $AnimatedSprite2D

func _ready() -> void:
    # Listen for when an animation finishes
    anim.animation_finished.connect(_on_animation_finished)

func _physics_process(delta: float) -> void:
    var dir := Input.get_axis("ui_left", "ui_right")
    velocity.x = dir * 200.0
    move_and_slide()
    _update_animation(dir)

func _update_animation(dir: float) -> void:
    if not is_on_floor():
        _play("jump")
    elif dir != 0:
        _play("run")
        anim.flip_h = dir < 0   # mirror sprite when moving left
    else:
        _play("idle")

# Guard: only call play() if animation is changing
# prevents restarting the same animation every frame
func _play(name: String) -> void:
    if anim.animation != name:
        anim.play(name)

# One-shot animations (attack, hurt) — resume idle when done
func play_attack() -> void:
    anim.play("attack")   # animation_finished fires when done

func _on_animation_finished() -> void:
    if anim.animation in ["attack", "hurt"]:
        anim.play("idle")   # return to idle after one-shot plays
